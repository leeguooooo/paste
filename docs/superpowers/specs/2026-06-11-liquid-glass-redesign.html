Pastyx Liquid Glass 玻璃岛改造

设计文档 · 2026-06-11 · 范围：apps/macos（主面板 + 设置页） · 状态：已获用户批准

1. 目标与非目标

目标：将 macOS 端 UI 从"全宽透明条带 + 不透明深色托盘"改造为 Apple Liquid Glass 风格的悬浮玻璃岛：居中贴底的圆角毛玻璃面板，真实透出桌面/全屏应用内容；卡片采用半透深底 + 类型色顶边；设置页收进岛内。

非目标：不追求 macOS 26 原生 Liquid Glass 的折射/流动效果（Electron 无 API）；不改变任何交互逻辑（热键、粘贴、搜索、收藏、同步）；不引入 CSS backdrop-filter（性能教训，2026-06-11 已全数移除）。

2. 窗口层（main.cjs）

项现状改为
窗口形态全宽 480px 底部条带，transparent: true岛即窗口：宽 min(1040, workArea.width × 0.86)，高 ≈ 420px，水平居中，底边距 16px
玻璃材质无（CSS 透明）vibrancy: "under-window" + visualEffectState: "active"，移除 transparent/backgroundColor，roundedCorners: true
保留不动type: "panel"、setAlwaysOnTop(screen-saver)、visibleOnAllWorkspaces({visibleOnFullScreen})、blur→hide、单实例锁、backgroundThrottling: false、光标所在屏定位
设置展开机制window:set-expanded IPC 把窗口撑满工作区整套移除（IPC、preload 暴露、App.tsx 调用、hide/crash 复位逻辑），设置在岛内渲染
页面背景必须保持透明（body { background: transparent }），vibrancy 材质才会透上来；玻璃感由系统合成器渲染，渲染进程零额外开销。

3. 渲染层（index.css / App.tsx）

  • 布局：app-shell 即岛的内容区。顶行：玻璃药丸搜索框（占主宽）+ 星标按钮 + 设置按钮（收进岛内，不再悬浮窗外）。下方横向卡片流（350px 高度区），底部留呼吸边距。
  • 卡片（已选方案 C）：深色 rgba(30,30,38,0.55) / 浅色 rgba(255,255,255,0.6)；0.5px 白/黑描边 + inset 0 1px 0 顶部高光；2px 类型色顶边沿用 getTypeAccent 配色（文本蓝 / 代码紫 / 图片绿 / 链接橙）；类型 pill 同色染色（色字 + 12–18% 同色底）。卡片不加任何 backdrop-filter。
  • 选中/悬停：仅 transform（translateY/scale）+ 边框色过渡；选中态边框换类型色、不透明度提升。阴影固定不参与过渡。
  • 深浅色：沿用 prefers-color-scheme 双套变量；vibrancy 材质自动跟随系统。

4. 设置页

  • 移除全屏蒙层 + 冻结截图背板机制（captureWindow 背板路径、settingsBackdropDataUrl 状态、相关 CSS）。
  • 设置成为岛内视图：进入时卡片区滑出、设置视图滑入（transform 过渡），玻璃分段 Tab（Sync / Account / System），内容区在岛内滚动；Esc / 关闭按钮返回卡片区。窗口尺寸不变。
  • 托盘入口不变：托盘菜单 Settings → showSettings() 已有"显示窗口 + 广播 ui:open-settings"链路（main.cjs / App.tsx onOpenSettings），岛内设置视图直接复用该事件，无需新 IPC；同理入场动画复用已存在的 window:shown 事件（preload 已暴露），改动只在渲染层。

5. 动效

  • 入场每次重放：监听 window:shown 重置动画类，岛内容 translateY(24px)+opacity 0→1，约 200ms（现 springUp 只在挂载时播一次，等于没有动画）。
  • 所有动效仅 transform/opacity；backgroundThrottling: false 已保证首帧流畅。

6. 风险与回退

风险处置
vibrancy 与 type: panel / 全屏可见组合在 Electron 37 的兼容性未经实测实现的第一步先做最小窗口实验验证；若冲突，回退为纯 CSS 半透岛（布局/卡片不变，失去透桌面效果），窗口参数一行切换
圆角为系统标准半径（约 10px），小于 mockup 的 22px已知妥协，用户已知情接受；Electron 无自定义半径 API
移除 expanded 机制牵连今天刚加的 hide/crash 复位逻辑连根移除而非留壳，避免死代码；行为由"设置常驻岛内"天然覆盖

7. 验证

  • 现有 28 个单测不受影响（纯 UI 改造，不触存储/同步）；typecheck + vite build 必须干净。
